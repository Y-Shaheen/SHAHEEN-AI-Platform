## 🌌 About

**SHAHEEN-AI-Platform** is a free, open-source, enterprise-grade Agentic AI framework that brings the power of large language models to every major messaging platform. Built on Python 3.12 and FastAPI, it provides a unified layer to deploy intelligent chatbots and autonomous agents across Telegram, Discord, Slack, WeChat, QQ, DingTalk, Lark, and 15+ more platforms — all from a single, beautiful web dashboard.

Whether you are building a personal AI companion, an enterprise knowledge base assistant, an autonomous workflow agent, or a customer service bot, SHAHEEN-AI-Platform gives you the infrastructure to do it reliably and scalably.

## 📐 System Design

- **Event-driven core** — all platform messages flow through a unified event bus
- **Adapter pattern** — each messaging platform is an isolated adapter; new ones are easy to add
- **Provider abstraction** — LLMs are pluggable; switch with zero code changes
- **Plugin hooks** — plugins intercept events before and after LLM responses
- **Async-first** — built on Python `asyncio` for high concurrency
- **SQLModel ORM** — SQLite for lightweight deployments, PostgreSQL for production
- **Agent Sandbox** — isolated subprocess with resource limits for safe code execution

---

## 🔄 Workflow

```
User Message (any platform)
        │
        ▼
Platform Adapter (Telegram / Discord / QQ / ...)
        │
        ▼
Event Bus → Plugin Pre-Hooks
        │
        ▼
Agent Runtime
  ├── Context Manager (history, persona, compression)
  ├── Tool Dispatcher (web search, code exec, MCP, ...)
  └── LLM Provider (OpenAI / Anthropic / Gemini / ...)
        │
        ▼
Plugin Post-Hooks → Response Formatter
        │
        ▼
Reply → Platform Adapter → User
```

## ☁️ Cloud Deployment

### 🚂 Railway (Recommended)

Railway is the easiest way to deploy SHAHEEN-AI-Platform with zero server management.

**Step 1 — Create a Railway project**
1. Go to [railway.app](https://railway.app) and sign in
2. Click **New Project** → **Deploy from GitHub repo**
3. Select your fork of `Y-Shaheen/SHAHEEN-AI-Platform`

**Step 2 — Add services**
1. Click **+ New** → **Database** → **Add PostgreSQL**
2. Click **+ New** → **Database** → **Add Redis**

**Step 3 — Configure environment variables**

In your service settings → Variables, add:

```
DATABASE_URL=${{Postgres.DATABASE_URL}}
REDIS_URL=${{Redis.REDIS_URL}}
SESSION_SECRET=your-secure-random-string
OPENAI_API_KEY=sk-...
```

Add any additional LLM API keys from the [Environment Variables](#-environment-variables) section.

**Step 4 — Configure volumes**

1. Go to your service → **Volumes**
2. Add a volume mounted at `/AstrBot/data` to persist your configuration and database

**Step 5 — Deploy**
1. Click **Deploy** — Railway will build and start your container automatically
2. Go to **Settings** → **Networking** → **Generate Domain** for a public URL

**Step 6 — Configure Health Check**

In Railway service settings:
- **Health Check Path:** `/`
- **Health Check Timeout:** `30s`

**Step 7 — Auto Deploy**

Enable **Auto Deploy** in your service settings to automatically redeploy when you push to your GitHub repo.

**Step 8 — Custom Domain**
1. Go to **Settings** → **Networking** → **Custom Domain**
2. Add your domain and follow the DNS instructions

**Step 9 — Monitoring**
- Railway provides built-in metrics: CPU, memory, network
- View real-time logs from the **Deployments** tab

2. Connect your GitHub repo
3. Set **Build Command:** `pip install -r requirements.txt`

## 🗄 Database

SHAHEEN-AI-Platform uses SQLModel (SQLAlchemy + Pydantic) for all database operations.

### SQLite (Default)

No configuration needed. Data is stored in `data/astrbot.db`.

```yaml
# data/config.yaml
database:
  type: sqlite
  path: data/astrbot.db
```

```bash
# Set environment variable
DATABASE_URL=postgresql+asyncpg://user:password@host:5432/dbname
```

### MySQL / MariaDB

```bash
DATABASE_URL=mysql+aiomysql://user:password@host:3306/dbname
```

### Redis (Caching & Sessions)

```bash
REDIS_URL=redis://:password@host:6379/0
```

Redis is optional but recommended for production deployments to share session state across multiple instances.

## 🧠 Machine Learning Pipeline

```
Input (text / image / audio)
        │
        ▼
Preprocessing (format detection, transcription for audio)
        │
        ▼
Context Assembly (history + persona + knowledge base results)
        │
        ▼
RAG Retrieval (FAISS vector search + BM25 keyword ranking)
        │
        ▼
LLM Inference (streaming, function calling, MCP tools)
        │
        ▼
Post-processing (markdown rendering, TTS if needed)
        │
        ▼
Response Delivery
```

**Vector Store:** FAISS (local) with optional remote backends  
**Embedding models:** OpenAI `text-embedding-3-small/large`, local Ollama models  
**Retrieval:** Hybrid BM25 + dense vector search via `rank-bm25` + FAISS

```python
from astrbot.api.star import Star, register
from astrbot.api.event import filter, AstrMessageEvent

@register("my_plugin", "My Plugin", "1.0.0", "A custom plugin")
```
